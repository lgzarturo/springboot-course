# Sistema de Reservas de Hotel con TemГЎtica PokГ©mon

> Arquitectura Hexagonal + Screaming Architecture + TDD + Spring Boot 3 + Kotlin

## рҹҺҜ VisiГіn General

Desarrollar un sistema de reservas de hotel con temГЎtica PokГ©mon que permita:

- вң… GestiГіn completa de usuarios (entrenadores PokГ©mon)
- вң… GestiГіn de habitaciones temГЎticas por tipo PokГ©mon
- вң… Proceso de reservaciГіn con carrito de compras
- вң… Sistema de pagos integrado
- вң… GamificaciГіn: puntos, experiencia y desbloqueo de PokГ©mon
- вң… AutenticaciГіn con JWT (usuarios y administradores)
- вң… ReseГұas y servicios adicionales
- вң… CRUDs completos para todas las entidades

## рҹҸ—пёҸ Arquitectura del Sistema

Principios Aplicados:

1. Screaming Architecture: La estructura "grita" el propГіsito del negocio
2. Hexagonal Architecture: Dominio independiente, puertos y adaptadores
3. SOLID: Principios aplicados en cada capa
4. TDD: Tests primero, luego implementaciГіn
5. DDD: Domain-Driven Design para la lГіgica de negocio

## Multiplicidades

Es la forma en que se relacionan las entidades en la base de datos.

- User вҶ’ Reservation - 1:N - Un usuario puede tener mГәltiples reservas
- Room вҶ’ Reservation - 1:N - Una habitaciГіn puede tener mГәltiples reservas
- Reservation вҶ’ Payment - 1:1 - Una reserva tiene un pago
- User вҶ’ Pokemon - 1:N - Un usuario puede tener mГәltiples PokГ©mon
- Reservation вҶ’ Review - 1:1 - Una reserva puede tener una reseГұa
- Reservation вҶ” Service - N:M - Reservas pueden tener mГәltiples servicios
- User вҶ’ TrainerProfile - 1:1 - Un usuario tiene un perfil de entrenador
- User вҶ’ Cart - 1:1 - Un usuario tiene un carrito activo
- Cart вҶ’ CartItem - 1:N - Un carrito tiene mГәltiples items
